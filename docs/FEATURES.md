# Vibric 기능 명세

> **문서 버전**: 1.1  
> **최종 수정일**: 2024-12-24

---

## 1. 기능 개요

Vibric은 4개의 주요 탭으로 구성된 바이브코딩 빌더입니다.

| 탭 | 상태 | 설명 |
|-----|------|------|
| **Canvas** | MVP | Figma 스타일 시각적 편집기 |
| **Code** | MVP | Monaco 기반 코드 에디터 |
| **Assets** | Phase 2 | 에셋 관리 (로고, 폰트, 이미지) |
| **Dashboard** | Phase 2 | 프로젝트 설정 및 관리 |

### 2.2 사용자 시나리오

**시나리오 1: 페이지 시각적 편집**
1. AI가 생성한 페이지가 캔버스에 표시됨
2. 사용자가 요소 클릭 → 바운딩 박스 표시
3. 드래그로 위치 변경 → 코드 자동 업데이트
4. 모서리 드래그로 크기 변경 → 코드 자동 업데이트

**시나리오 2: 선택 모드 사용** ✅ 구현 완료
1. 하단 툴바의 "선택" 버튼 클릭 → 선택 모드 활성화
2. 요소 호버 시 파란색 하이라이트 표시
3. 요소 클릭 시 선택 박스 + AI 수정 입력창 표시
4. AI 입력창에 수정 요청 입력 → AI가 코드 수정

**시나리오 3: 페이지 패널 사용** ✅ 구현 완료
1. 우측 Pages 패널에서 현재 페이지 목록 확인
2. 페이지는 프로젝트의 실제 HTML/TSX/JSX 파일에서 자동 감지
3. 페이지 클릭 시 해당 페이지로 프리뷰 전환

### 4.2 Artifact 형식

AI가 코드를 생성할 때 사용하는 XML 태그 형식:

```xml
<boltArtifact id="unique-id" title="설명">
  <boltAction type="file" filePath="src/components/Button.tsx">
    // 코드 내용
  </boltAction>
  <boltAction type="shell">
    npm install some-package
  </boltAction>
</boltArtifact>
```

### 4.3 AI 프롬프트 시스템

**시스템 프롬프트 구성요소:**
1. 기본 지시사항 (역할, 규칙)
2. Artifact 형식 규칙
3. 현재 파일 시스템 상태
4. 선택된 캔버스 요소 정보 (수정 요청 시)
5. 사용자 프로젝트 에셋 정보 (Phase 2)
